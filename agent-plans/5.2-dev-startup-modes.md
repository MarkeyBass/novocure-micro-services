# 5.2 — Dev Startup Modes: Local vs Docker Compose

Two supported ways to run the stack. Pick one per session — don't mix them unless you know which ports are in use.

---

## Mode A — Local API dev (recommended for iterative development)

Infrastructure (databases, RabbitMQ) runs in Docker. APIs and Angular run on the host.

**Best for:** fast iteration — `dotnet watch` hot-reloads code changes in seconds.

### What runs where

| Service | Host | How |
|---|---|---|
| MongoDB | `localhost:27018` | Docker |
| SQL Server | `localhost:1433` | Docker |
| RabbitMQ | `localhost:5672` | Docker |
| HousingApi | `http://localhost:5125` / `https://localhost:7152` | `dotnet watch` |
| TodoApi | `http://localhost:5188` / `https://localhost:7236` | `dotnet watch` |
| Angular | `http://localhost:4200` | `npm run start:local` |

### Commands

**1. Start infrastructure only**

```bash
cd med-tech-local-infra
docker compose up -d mongo sqlserver rabbitmq
```

**2. Start HousingApi** (new terminal, from `HousingApi/`)

```bash
dotnet watch --launch-profile http
```

**3. Start TodoApi** (new terminal, from `TodoApi/`)

```bash
dotnet watch --launch-profile http
```

Use the `http` profile to avoid SSL certificate issues with the Angular proxy. The `https` profile also works — just make sure the proxy targets in `proxy.local.json` match.

**4. Start Angular** (new terminal, from `client/`)

```bash
npm run start:local
```

### How configuration is resolved

`appsettings.json` in each API already points everything to `localhost`:

- `RabbitMQ.HostName` → `localhost`
- `HousingDatabase.ConnectionString` → `mongodb://...@localhost:27018/...`
- `ConnectionStrings.TodoContext` → `Server=localhost,1433;...`

No overrides needed — the local defaults are correct for this mode.

## Mode B — Full Docker Compose

Everything runs in containers. No local .NET processes needed.

**Best for:** AI-agent-driven development where changes per iteration are large, or verifying the production-like container environment.

**Trade-off:** each code change requires a full image rebuild (~60–120s per service).

### What runs where

| Service | Host port | Container port | How |
|---|---|---|---|
| MongoDB | `27018` | `27017` | Docker |
| SQL Server | `1433` | `1433` | Docker |
| RabbitMQ | `5672` / `15672` | `5672` / `15672` | Docker |
| HousingApi | `7152` | `8080` | Docker (built from `HousingApi/Dockerfile`) |
| TodoApi | `5188` | `8080` | Docker (built from `TodoApi/Dockerfile`) |
| Angular | `4200` | — | `npm run start:compose` (host) |

Angular still runs on the host — running it in Docker adds friction with no benefit during dev.

### Commands

**1. Build and start everything**

```bash
cd med-tech-local-infra
docker compose up -d --build
```

`--build` forces Docker to rebuild changed images. Omit it if only restarting without code changes.

**2. Rebuild a single service after changes** (faster than rebuilding all)

```bash
docker compose up -d --build housing-api
# or
docker compose up -d --build todo-api
```

**3. Start Angular** (from `client/`)

```bash
npm run start:compose
```

**4. View logs**

```bash
docker logs -f housing-api
docker logs -f todo-api
```

### How configuration is resolved

Docker Compose injects environment variables that override `appsettings.json` at runtime:

| Variable | Value | Overrides |
|---|---|---|
| `ASPNETCORE_URLS` | `http://+:8080` | Forces HTTP-only inside the container |
| `RabbitMQ__HostName` | `rabbitmq` | Replaces `localhost` with the compose service name |
| `HousingDatabase__ConnectionString` | `mongodb://...@mongo:27017/...` | Uses `mongo` service name |
| `ConnectionStrings__TodoContext` | `Server=sqlserver,1433;...` | Uses `sqlserver` service name |

The `__` double-underscore maps to nested JSON keys (`RabbitMQ.HostName`). This is standard ASP.NET Core configuration behaviour — no code changes needed when switching modes.

## Angular proxy: how environment switching works

The Angular service files use relative URLs (`/api/locations`, `/api/TodoItems`). The Angular dev server proxy resolves where those go.

| Command | Proxy config | HousingApi target | TodoApi target |
|---|---|---|---|
| `npm run start:local` | `proxy.local.json` | `http://localhost:5125` | `http://localhost:5188` |
| `npm run start:compose` | `proxy.compose.json` | `http://localhost:7152` | `http://localhost:5188` |

`start:local` targets the `http` launch profile port (5125) because the `https` port (7152) uses TLS and the Vite proxy would get `EPROTO` / "socket hang up" trying to connect to it over plain HTTP.

## Switching between modes

Stop whatever is running before switching.

**Stop compose services:**

```bash
cd med-tech-local-infra
docker compose down
```

**Stop a specific service only** (e.g. to replace it with a local `dotnet watch`):

```bash
docker compose stop housing-api
```

`docker compose down` without `-v` preserves database volumes, so data survives restarts.

## Quick reference

| Goal | Command |
|---|---|
| Start infra only (Mode A) | `docker compose up -d mongo sqlserver rabbitmq` |
| Start full stack (Mode B) | `docker compose up -d --build` |
| Rebuild one API (Mode B) | `docker compose up -d --build housing-api` |
| Angular → local APIs | `npm run start:local` (from `client/`) |
| Angular → compose APIs | `npm run start:compose` (from `client/`) |
| Tail housing-api logs | `docker logs -f housing-api` |
| Stop all containers | `docker compose down` |
