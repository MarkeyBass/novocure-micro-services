Run the API with HTTPS (uses the `https` profile from Properties/launchSettings.json):

  dotnet run --launch-profile https

Short form:

  dotnet run -lp https

Run this from the project directory (the folder containing BookStoreApi.csproj).

MongoDB (Docker Compose in local-databases/docker-compose.yaml)

  The compose file sets MONGO_INITDB_ROOT_USERNAME / MONGO_INITDB_ROOT_PASSWORD for the
  container only. appsettings.json does NOT expand ${VAR} like Docker Compose — use a
  real URI in configuration, or override from the shell:

    export BookStoreDatabase__ConnectionString='mongodb://admin:password123@localhost:27018/?authSource=admin'

  (Double underscore __ is how nested keys map to BookStoreDatabase:ConnectionString.)

  Port 27018 is the host port published from the container; credentials must match
  the environment block under the mongodb service in docker-compose.yaml.
